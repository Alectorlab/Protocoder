Getting started

MakeWithMoto is a prototyping tool to help you prototype hardware projects easily through the Android platform using javascript in less than a minute. This tutorial is intended on serving as a walk-through of how to get started. So strap in and hang on.

The layout of the builder page contains an editor pane, a console log, and a quick-search documentation panel. From here, you can build a fully functioning hardware project written entirely in javascript, including native android interface elements, such as buttons, slider bars and more.

Adding UI elements

Add a button

MakeWithMoto has several elements that are available to create a view to interact with. Let's create a Button. To add one, simply use the method moto.addButton(String, Function). 


  moto.addButton("Label", function() {
    // This will be fired when the button
    // has been pressed
  });
Add a toggle button

A toggle button allows the user to change a setting between two states. A toggle button is used to determine between states, such as should an LED be turned on or off. 


  moto.addAToggleButton("A label", function(on) {
    // on will say if the toggle is on
    // or not
  });
Add a seekbar

A SeekBar is an extension of ProgressBar that adds a draggable thumb. The user can touch the thumb and drag left or right to set the current progress level or use the arrow keys. It allows you to "slide" values from 0 to a max element. 


  moto.addSeekBar("Slide me", 200,
                10, function(newValue) {
  });
Add a text field

Displays text to the user. This is simply a label that contains text to show the user information. 


  moto.addTextLabel("A label");
Add a checkbox

Add a two-state button that can either be checked or unchecked.

This is useful if you want a function to happen only under certain conditions. 


  moto.addACheckbox("A label", 
                true, 'nameOfElement');
Show a toast

A toast provides simple feedback about an operation in a small popup. It only fills the amount of space required for the message and the current activity remains visible and interactive. 


  moto.showToast("A label", 20);

Creating components

Components are the core driver for building against hardware. These are how we'll interact with the components on our hardware. The syntax of creating these are special. In order to create one, in every case we'll give it a name that we'll refer to when using it.

Create a digital output

A digital output is the most basic component available. LEDs are an example of a digital output. 


  // pin 0 is the onboard LED, for example
  moto.createDigitalOutput(0, 'onBoardLED');
Turn on a digital output

To turn on a digital output device, simply pass it `false` as we're only setting the devices to pull down devices. 


  // moto.createDigitalOutput(0, 'onBoardLED');
  onBoardLED.write(false);
Turn off a digital output

To turn off a digital output device, simply pass it `true`, as shown below 


  // moto.createDigitalOutput(0, 'onBoardLED');
  onBoardLED.write(true);
Create a new digital input

To create a new digital input device, such as a push button or a touch sensor, simply call the method createDigitalInput: 

moto.createDigitalInput(9, 'digitalInput', 
                              function() {});
To read off the digital input

To read the current value of the digital input device, simply call the function `read` within a callback as show here. The current value of the digital input will be passed into the callback function such as: 


  // moto.createDigitalInput(9, 'digitalInput', 
  //        function() {});
  digitalInput.read(function(v) {
    // v is the current value of the device
  });
To trigger an event from the device

Sometimes it's useful to trigger an event when the digital input device goes low or high. To do this, we can use the function `when` 

// moto.createDigitalInput(9, 'digitalInput', 
  //        function() {});
  digitalInput.when(false, function() {
    // fired when the digitalInput goes low
  });
To repeatedly trigger an event from the device

Sometimes it's useful to trigger an event when the digital input everytime the device goes low or high. To do this, we can use the function `whenever` 

// moto.createDigitalInput(9, 'digitalInput', 
  //        function() {});
  digitalInput.whenever(false, function() {
    // fired when the digitalInput goes low
  });
Create an analog input

Analog inputs are useful for measuring things like voltage that can tell us interesting things about the surrounding world. Temperature sensors and potentiometer are good examples of analog inputs: 

moto.createAnalogInput(41, 'potentiometer', 
  function() {
    // Now we have an analog input
  });
Read an analog input

To read the current value of the analog input, use the method: read.


moto.createAnalogInput(41, 'potentiometer', 
  function() {});
  potentiometer.read(function(v) {
    moto.log("Current value is: " + v);
  });
To create a pwm output device

A Pulse Width Modulation (PWM) signal is a kind of a digital output, which has a certain structure in its temporal behavior. Specifically, the signal has a fixed period. Every time a period starts, the signal moves from LOW to HIGH. This is for things like servos. We create one of these passing the pin, then the frequency at which the device runs at, a common name we want to pass it and finally a callback 

moto.createPwmOutput(5, 50, 'pwm', function() {
    // Now you have a pwm module
    alert(value);
    pwm.setPulseWidth(value);
  });
To set the duty cycle

Setting the duty cycle of a pwm device is super easy. Simply call the method `setDutyCycle` with the new cycle of between 0 and 1, such as: 

//moto.createPwmOutput(5, 50, 'pwm', 
  //      function() {});
  pwm.setDutyCycle(0.5);
To set the pulse width

Setting the pulse width of a pwm device is super easy too. Simply call the method `setPulseWidth` with the new width. This usually ranges from 1000 to 2000 for typical servos. 

//moto.createPwmOutput(5, 50, 'pwm', 
  //      function() {});
  pwm.setPulseWidth(1500);
To create a pulse input device

One very common type of digital signals is one in which the information lies in the duration between changes in the signal level. Perhaps the simplest case to imagine is a magnetic or optical sensor sensing a turning shaft: whenever the shaft gets to a certain rotation angle, the sensor's output will change from LOW to HIGH, and at a different angle it will change back to HIGH. The resulting waveform whenever the shaft is constantly turning is a set of pulses. The faster the rotation, the narrower and closer together the pulses. By measuring the duration between the beginning of one pulse to the beginning of the next, we can know the amount of time it took the shaft to complete exactly one turn or, in other words, if we measure the frequency of the pulses, we can know the speed of turning. 

moto.createPulseInput(34, 'pi', function() {
    // Now you have a pi module
  });
To get the duration between signals

Getting the pulse input duration between signals. This will return the duration of the latest pulse in float form back to your callback function. 

//moto.createPulseInput(34, 'pi', 
  // function() {});
  pwm.getDuration(function(d) {
    // d is the duration as a float
  });
To get the frequency of signals

Getting the pulse input frequency of the signals. This will return the frequency of the latest pulse in float form back to your callback function. 

//moto.createPulseInput(34, 'pi', 
  // function() {});
  pwm.getFrequency(function(f) {
    // f is the frequency returned as a float
  });
To create a uart device

Universal Asynchronous Reception and Transmission (UART) is a very common, simple and useful serial communication interface. It serves as a basis for many ubiquitous protocols such as RS-232 (used on COM ports of old computers), and MIDI. Its basis is a one-way communication channel, on which one end transmits and the other end receives on a single wire. The basic data units transferred are 8-bit bytes. Each byte is sent on the wire bit-by-bit, preceded by a start bit which is simply the bit '0' and followed by an optional parity bit (used for error correction, but commonly not used at all) and one or two stop bits (commonly only one), which are simply the bit '1'. 

moto.createUart(34, 35, 9600, 'uart', function() {
    // Now you have a uart module
  });
To read from the uart device

To read from the uart device, we'll use the method `read` on it. This will simply take the bytes it reads from the input device, convert it to a string and return it back to the callback function. 

//moto.createUart(34, 35, 9600, 'uart', 
  // function() {});
  pwm.read(function(str) {
    // str is the string read
  });
To write to the uart device

Writing to the uart device is just as easy as writing. Using the method `write`, we can write a string to the device. 

//moto.createUart(34, 35, 9600, 'uart', 
  // function() {});
  pwm.write("hello", function(boolean) {
    // boolean will be true if successful
    // and will be false if unsuccessful
  });
To create a Two Wire device

Two Wire Interface (TWI) is the generic name for the interfaces commonly known as I²C and SMBus. It is an interface which enables half-duplex serial communication between multiple devices, sharing the same bus, with as little as 2 wires (hence the name). Each of the devices can be either master or slave, where masters are the only ones that can initiate data transactions. Each slave device has a unique address on the bus, which is used to select it as the target of a transaction. TWI supports multiple masters on a bus, but this is a rather rare use-case, and for the rest of this discussion we'll assume there is only one. 

moto.createTwoWire(pin, 'twi', function() {
  // We now have an Two wire device
});
To write and read to the Two Wire interface

We can write and read to the Two Wire using a simple `readWrite` function. 

//moto.createTwoWire(pin,'twi',function(){});
var address = 0,
    bytesToRead = 4,
    dataToWrite = ['0x01', '0x02', '0x03'];
    
twi.writeRead(address, dataToWrite, 
      bytesToRead, function(resp) {
  // The bytes read will be the response
});
To create a Serial Peripheral Interface

Serial Peripheral Interface (SPI) is a common interface used to communicate between integrated circuits. A single SPI bus enables a single master device to communicate with one or more slave devices at typical rates of up to tens of Mbit/sec. SPI is a full-duplex interface, meaning that transmission and reception can are taking place concurrently. SPI requires 3 wires shared by the master and all slaves and an additional wire between the master and each slave. 

moto.createSpi(misoPin, mosiPin, clockPin, 
            slavePins, 'spi', function() {
  // We now have an SPI
});
To write and read to the Spi device interface

We can write and read to the Serial Peripheral Device using a simple `readWrite` function. 

//moto.createSpi(misoPin, mosiPin, clockPin, 
  slavePins, 'spi', function() {});
spi.writeRead(0, ['0x01', '0x02', '0x03'], 
      function(resp) {
  // the resp will be the response
});
To vibrate the phone

You can vibrate the phone easily by using the method vibrate(num) and passing in the number of milliseconds to vibrate the phone. 

moto.vibrate(1000)
